## Key Architecture Decisions

1. **Parquet is always current.** `sync_from_odoo()` overwrites it on every "Mulai Sesi". Cannot use as price-change baseline.

2. **Mail tracking `old_value_float` is the truth.** The price BEFORE a change lives only in tracking. `detect_changes_since` uses this as baseline, not parquet.

3. **Two Odoo models for `list_price` tracking:** `product.product.list_price` (mass import / variant-level changes) AND `product.template.list_price` (UI/Update Harga page changes). Both must be queried.

4. **`product.template` tracking ‚Üí variant mapping.** Template-level tracking entries have `res_id` = template_id. A separate RPC maps template_id ‚Üí variant_ids via `product.product.product_tmpl_id`.
